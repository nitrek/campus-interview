DBMS

Easy: What is the difference between SQL and NoSQL databases?
SQL (Relational Databases): They store data in structured tables with rows and columns. Predefined schema, use SQL. ACID guarantees. Examples: MySQL, PostgreSQL, Oracle. NoSQL (Non-relational Databases): Store data as document, key-value, wide-column, or graph. Flexible schema. BASE model. Examples: MongoDB, Redis, Cassandra. Hint: Think "structure vs. flexibility." Follow-up: When would you choose NoSQL over SQL?
Easy: What are primary keys and foreign keys?
Primary Key: Uniquely identifies a record. Unique values, no NULL, only one per table. Foreign Key: Links tables. Field in child table referring to parent table primary key. Hint: Primary key = student's ID. Foreign key = student's ID in Enrollments table.
Easy: Difference between DELETE, TRUNCATE, DROP in SQL?
DELETE: Removes rows based on WHERE. Rollback possible. Slower. TRUNCATE: Removes all rows. No WHERE. Cannot rollback in most DBs. Fast. DROP: Removes entire table structure and data. Cannot undo.
Medium: What is database normalization? Explain 1NF, 2NF, 3NF.
Normalization organizes tables/columns to minimize redundancy. 1NF: Atomic values, unique records. 2NF: 1NF + all non-key attrs depend on entire primary key. 3NF: 2NF + no transitive dependencies. Hint: 1NF: No multi-value cells. 2NF: No partial dependencies. 3NF: No transitive. Follow-up: What is denormalization and when would you use it?
Medium: What are database indexes and why are they important?
Indexes speed up data retrieval. Pointer to table data, like a book index. Hint: Phone book analogy. Follow-up: Clustered vs Non-clustered index?
Medium: What are ACID properties?
Atomicity: All or nothing. Consistency: Valid state transitions. Isolation: Transactions don't interfere. Durability: Committed data persists. Follow-up: How is isolation implemented? (e.g., locking)
Difficult: Explain transaction isolation levels.
Read Uncommitted: Dirty reads possible. Read Committed: Prevent dirty reads, non-repeatable possible. Repeatable Read: Prevent dirty/non-repeatable, phantom possible. Serializable: Fully isolated, serial execution. Follow-up: Optimistic vs pessimistic locking?
Difficult: Database sharding vs partitioning.
Sharding: Horizontal split across DB servers. Each shard independent. Partitioning: Split table into smaller pieces on same server. Hint: Partitioning = single node, sharding = multi-node. Follow-up: Common sharding strategies?
Difficult: N+1 query problem?
Occurs when N queries are made for related data instead of one join query. Solved by eager loading, JOIN, or IN clause. Follow-up: When is JOIN less performant than IN clause?

Java

Easy: Difference between JDK, JRE, JVM?
JVM: Executes Java bytecode. Abstract machine. JRE: JVM + libraries to run programs. JDK: JRE + development tools to write programs. Hint: JVM = core, JRE = runtime, JDK = development.
Easy: Difference between String, StringBuilder, StringBuffer?
String: Immutable. StringBuilder: Mutable, not thread-safe, faster. StringBuffer: Mutable, thread-safe, slower.
Easy: Interface vs Abstract class
| Feature | Abstract Class | Interface | | Methods | Can have both abstract and concrete | Only abstract (Java 8+ default/static allowed) | | Variables | Any kind | Only public static final | | Inheritance | Single | Multiple | | Constructor | Yes | No | | Purpose | Common base | Contract for unrelated classes |
Medium: Java memory model (Heap vs Stack)
Stack: Method calls, primitives, references. LIFO. Fast. Limited. Heap: Objects, dynamic memory. Shared. Slower. Managed by GC.
Medium: Checked vs Unchecked exceptions
Checked: Compile-time, must handle. (IOException) Unchecked: Runtime, optional. (NullPointerException) Follow-up: finally block?
Medium: equals() & hashCode() contract
Equal objects must have equal hash codes. Override both for HashMap/HashSet. Follow-up: What happens if only equals() is overridden?
Difficult: Garbage Collection
Mark & Sweep: Mark reachable objects, sweep unreachable. Compact optional. Modern GCs use advanced algorithms. Follow-up: Name GC types (Serial, Parallel, G1GC)
Difficult: Fail-fast vs Fail-safe iterators
Fail-fast: Throws exception on concurrent modification. Fail-safe: Works on clone, no exception. Follow-up: Why CopyOnWriteArrayList over synchronized ArrayList?
Difficult: HashMap internal working
Buckets (array + linked list/tree). hashCode -> index -> bucket -> equals() check. Resize when too full.

Node.js

Easy: What is Node.js?
Back-end JS runtime on V8, executes JS outside browser, server-side scripting.
Easy: What is NPM?
Node Package Manager. CLI + registry. package.json lists dependencies.
Easy: Purpose of package.json?
Manifest file: dependencies, scripts, metadata. Central for project reproducibility.
Medium: Explain event loop
Single thread non-blocking I/O. Kernel handles async ops, callbacks queued, event loop processes them.
Medium: require() vs import
require(): CommonJS, synchronous. import: ESM, asynchronous, modern standard.
Medium: Middleware in Express.js
Functions with req, res, next. Can modify request/response, end cycle, call next.
Difficult: Node.js streams
Readable, Writable, Duplex, Transform. Memory-efficient, process data in chunks. Follow-up: Flowing vs paused mode?
Difficult: Node.js cluster module
Fork multiple workers, share ports, scale across CPU cores.
Difficult: Error handling in async Node.js
Callbacks (error-first) or Promises/async-await. Handle uncaughtException and unhandledRejection at process level.

Python

Easy: List vs Tuple
List: mutable, [] Tuple: immutable, (). Tuples can be dict keys, memory-efficient.
Easy: Python decorators
Decorators add functionality without modifying object. @ syntax.
Easy: .py vs .pyc
.py = source code. .pyc = compiled bytecode, faster next run.
Medium: GIL
Mutex for one thread at a time in CPython. Blocks CPU-bound parallelism, allows I/O concurrency. Follow-up: multiprocessing module for true parallelism.
Medium: List comprehensions & generators
List comprehension: [x*x for x in range(10)] Generator: (x*x for x in range(10)), yields items one by one. Follow-up: yield keyword
Medium: __init__ vs __new__
__new__: creates object __init__: initializes object
Difficult: Memory management
Reference counting + cyclic GC. Zero references = deallocated.
Difficult: Metaclasses
Metaclass = class of class. Can modify class creation automatically.
Difficult: Descriptor protocol
__get__, __set__, __delete__. Underlies properties, methods. Follow-up: Data descriptor vs non-data descriptor.

React

Easy: JSX
JSX = HTML-like syntax in JS. Needs transpiling.
Easy: Props vs State
Props: from parent, read-only State: mutable, inside component, triggers re-render
Easy: Virtual DOM
VDOM = in-memory UI copy, diffing, minimal real DOM updates.
Medium: Lifecycle with hooks
useEffect manages mount, update, unmount. Dependency array controls execution. Cleanup function for unmount.
Medium: Hooks
useState, useEffect, useContext, useRef. Modern functional component state & lifecycle.
Medium: Prop drilling
Pass props through multiple levels. Avoid with Context API, state management libs, or composition.
Difficult: Reconciliation (diffing)
Diffing: different type = replace tree. Same type = update attributes. List keys crucial.
Difficult: useMemo & useCallback
useMemo = memoize value useCallback = memoize function Avoid expensive re-renders.
Difficult: HOC vs Custom Hooks
HOC wraps component, injects props. Custom hooks share logic inside functional components. Hooks preferred.

Be specific. Show problem-solving & technical depth. Follow-up: Alternative solutions considered?
